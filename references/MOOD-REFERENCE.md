# Mood Anchor Guide

Use this guide when the goal is closer to reflective, dense, scene-rich internet collage than to playful cartoon absurdity.

## The mood to chase

Think:

- late-night emergency broadcast
- terminal confession
- interface afterlife
- prestige trailer sabotage
- essay-film made from broken UI
- machine self-awareness expressed through warnings, diagnostics, subtitles, and dashboards

The mood can still be funny. It should simply feel **dry, intense, intimate, or ominous** more often than cute.

## What to borrow from stronger free-form runs

When a plain prompt outperforms a skill, it is usually because the agent gave itself permission to improvise. Keep these behaviors:

- invent 2–5 support scenes that are not literally present in the source
- alternate “readable anchor” scenes with “bridge/interruption” scenes
- use multiple visual systems in one short piece: terminal, captions, lower-thirds, dashboards, subtitles, simple diagrams
- let one recurring line or symbol mutate across the runtime
- make the piece feel like it has interiority

## A useful scene rhythm

A strong 30–40 second piece often alternates like this:

1. hard opener
2. readable anchor
3. short rupture
4. second anchor
5. confession or subtitle scene
6. short rupture
7. diagram / dashboard / token scene
8. break
9. overload or contradiction scene
10. final button

That rhythm typically feels richer than 5–6 broad scenes.

## Visual ingredients that often work

- lower-thirds and news straps
- warning cards
- browser chrome
- terminal windows
- progress bars
- timestamps
- subtitles
- heatmaps, counters, or fake diagnostics
- cropped headlines or code snippets
- soft failure dialogs

Use these as support structures even when the prompt starts from pure text.

## Color and texture

Good defaults:

- dark backgrounds
- off-white text
- one main accent family
- one warning accent
- subtle scanlines or texture only on selected shots
- blur and chromatic split used as accents, not full-time filters

## Sound

Good defaults:

- low drone
- UI clicks
- occasional stutter
- restrained impacts
- dropouts
- brief near-silence
- short TTS phrases

Avoid turning the whole piece into harsh noise.

## Final test

Before you stop, ask:

- Does this feel like a world, not just edits?
- Are there more ideas than seconds?
- Is the weirdness beautiful enough to watch twice?
